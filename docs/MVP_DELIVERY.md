# MVP 交付清单

当前主线面向“试点教学场景 MVP”交付，范围固定为：

- 单二进制 + 默认 SQLite 启动
- PC Web 前端可直接使用
- 首次 bootstrap 创建首个管理员
- 管理员前端创建用户、班级、课程，并完成教师分配与学生选课
- 教师前端创建模板、版本和实验任务
- 学生上传报告、截图、代码包或 Git 链接
- 系统完成基础解析、规则核查、LLM 初评
- 教师复核、发布，学生查看已发布结果
- 个人报告、实验统计、课程统计
- HTML / CSV / XLSX / PDF 导出

## 交付判断

满足以下条件即可视为 MVP 可交付：

1. 首个管理员可通过前端完成系统初始化。
2. 管理员可通过前端完成最小教学配置：
   - 创建 teacher / student 用户
   - 创建班级
   - 创建课程
   - 分配课程教师
   - 登记学生选课
3. 教师可通过前端完成最小教学任务配置：
   - 创建评价模板
   - 创建并发布模板版本
   - 创建并发布实验
4. 学生可上传成果并生成提交记录。
5. 教师可完成初评、复核、发布。
6. 学生可查看已发布评价。
7. 教师可导出 HTML / CSV / XLSX / PDF 报表。
8. `go test ./...`、前端构建、LoongArch 交叉编译通过。

## 本版仍保留的增强项

以下属于可交付后的增强，而不是阻塞 MVP 的条件：

- 截图 OCR 深度抽取
- 旧式 `.doc` 深度正文抽取
- 更完整的后台列表与搜索页
- 班级维度单独统计面板
- 批量样例压测与更细性能基线
- 更复杂的 PDF 模板、分页和图表布局
