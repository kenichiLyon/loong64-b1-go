# 安全与合规基线

## 数据安全

- 不提交真实学生数据、生产导出报表、API Key、Token、数据库密码或私钥。
- 云端 LLM 默认只接收脱敏摘要、评分规则和必要证据片段。
- `llm_call_logs` 只保存模型信息、Prompt 版本、输入摘要 hash、结构化输出和错误状态，不保存密钥、完整 Prompt 或原始学生文件。

## 上传安全

- 限制文件大小、数量、扩展名和 MIME 类型。
- 上传文件保存前计算 SHA-256，使用服务端生成的对象存储 key，不信任客户端文件名作为路径。
- 压缩包解析必须防止 Zip Slip、路径穿越、符号链接逃逸、超大解压和嵌套炸弹。
- 学生代码首版不在 API 服务进程中执行。
- Git 链接当前只登记 URL 和 commit 元数据；拉取必须进入独立 worker，并配置超时、资源限制和白名单。

## LLM 安全

- LLM HTTP 网关必须使用请求上下文和超时；HTTPS 内网证书应在系统信任链配置，禁止在代码中关闭 TLS 校验。
- 系统评分规则和教师配置的 Rubric 优先级高于上传内容。
- Prompt 必须防止学生文档中的指令覆盖评分规则。
- 输出必须通过结构化 schema 校验；失败结果不得自动发布。
- 评分建议必须保留证据引用、置信度和人工复核入口。

## 权限与审计

- 管理员、教师、学生分角色授权。
- 教师只能访问被授权课程和班级。
- 学生只能访问自己的提交和已发布评价。
- 上传、解析、核查、LLM 调用、教师改分、结果发布、报表导出必须写审计日志。

## 报表安全

- 报表导出记录操作者、筛选条件、生成时间、文件 hash 和对象存储 key。
- 导出文件访问必须校验权限。
- 临时文件必须定期清理，禁止把导出目录作为公开静态目录直接暴露。
